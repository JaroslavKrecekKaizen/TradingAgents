# Agentic collaboration and skill sync

## Purpose

This document defines how Claude Code and Codex execute the skills-based
TradingAgents pipeline, and the sync mechanism that prevents drift between them.

## Architecture

```
.claude/commands/*.md          <-- canonical source (edit here)
        |
        | scripts/sync_agentic_commands.py
        v
.agents/skills/source-command-*/SKILL.md   <-- generated (do not edit)
```

- Claude Code commands in `.claude/commands/*.md` are the single source of truth.
- Codex skills in `.agents/skills/source-command-*/SKILL.md` are generated by
  `scripts/sync_agentic_commands.py` and must not be edited directly.
- Both surfaces share the same Python data tools in `scripts/`.
- Both surfaces write decisions to the same Obsidian vault via filesystem MCP.
- `CLAUDE.md` is the canonical project context. `AGENTS.md` is a slim stub that
  points at `CLAUDE.md` and documents only Codex-specific differences.

## Development workflow

1. Edit the Claude command in `.claude/commands/<name>.md`.
2. Run `python3 scripts/sync_agentic_commands.py` to regenerate Codex skills.
3. Run `python3 scripts/sync_agentic_commands.py --check` to verify sync.
4. Commit the source command and generated skill together.

## Platform transforms

The sync script applies these transforms when generating Codex skills:

| Claude Code | Codex |
|-------------|-------|
| `.claude/commands/<role>.md` | `.agents/skills/source-command-<role>/SKILL.md` |
| `CLAUDE.md` | `AGENTS.md` |
| `Claude Code subagents` | `Codex subagents` |
| `Use the Agent tool to spawn subagents` | `Spawn subagents` |
| No skill wrapper | Codex frontmatter, heading, trigger line, `## Command Template` |

These are the only valid differences. Everything else - safety rules, role
prompts, data fetch commands, rating scales, output formats - must be identical.

## Invalid drift

Fix immediately if found:

- Safety rules differ between surfaces.
- A role prompt changes in one surface but not the other.
- Data fetch command names, arguments, or output expectations differ.
- Rating scales, confidence requirements, or risk requirements differ.
- Decision log format differs.
- Any file references point to paths that do not exist.

## Shared tool layer

The Python scripts in `scripts/` are the shared data tool layer:

| Script | Arguments | Output |
|--------|-----------|--------|
| `fetch_market_data.py` | `<ticker> <date>` | OHLCV, indicators, verified snapshot |
| `fetch_sentiment.py` | `<ticker> <date>` | News headlines, StockTwits, Reddit |
| `fetch_news.py` | `<ticker> <date>` | Ticker news, global macro news |
| `fetch_fundamentals.py` | `<ticker> <date>` | Financials, balance sheet, cash flow, income |

Both Claude Code and Codex invoke these via `.venv/bin/python scripts/<name>.py`.
Output goes to stdout.

## Roles

- **Claude Code**: primary development tool. Edit commands, run interactive
  analysis, debug skills, manage git workflow.
- **Codex**: execution overflow and batch runs. Runs the same pipeline via
  generated skills when Claude tokens are exhausted or for scheduled execution.
- **Human**: reviews decisions, manages portfolio, sets development priorities.
